🧊 Paso 1: La RAM, un casillero gigante

Imagina una pared llena de casilleros numerados (como los del gimnasio). Cada casillero se llama byte y guarda un número entre 0 y 255. Eso es la RAM.

💡 Metáfora:
Tu programa alquila algunos casilleros mientras se ejecuta. Cada variable es una etiqueta que pegas a uno o varios casilleros para recordar dónde dejaste la información.

Las letras también se guardan como números (código ASCII): 'A'=65, 'B'=66… Por eso dentro de la RAM todo son números.

❓ ¿Qué es una "dirección de memoria"?

📏 Paso 2: Cada tipo alquila un número distinto de casilleros

No todos los datos ocupan lo mismo. La variable conoce su "tipo" y eso determina cuántos bytes (casilleros) necesita:

  • Booleano (boolean): 1 byte. Guarda 0 (false) o 1 (true).
  • Entero (integer): normalmente 4 u 8 bytes.
  • Decimal (float): 4 u 8 bytes.
  • Cadena (string): 1 byte por cada carácter. "Hola" ocupa 4 bytes.
  • Carácter (char): un string de longitud 1. Ocupa 1 byte.

❓ Si la palabra "PHP" ocupa 3 bytes, ¿qué está almacenando realmente la RAM?

💡 Paso 3: Lógica – el idioma de las decisiones

Todo en programación se reduce a verdadero (true) y falso (false). Los operadores AND, OR, NOT combinan estos valores.

NOT (!) es el más simple: invierte el valor. Si algo es verdadero, NOT lo vuelve falso, y viceversa. En PHP se escribe con el signo ! delante de la condición. Por ejemplo, si tienes $llueve = true;, entonces !$llueve es false. "No llueve" es la negación de "llueve".

📋 Paso 4: Algoritmos, las recetas del programador

Un algoritmo es una secuencia ordenada de pasos precisos. Ejemplo: preparar un vaso de agua.

1. Coger un vaso limpio
2. Abrir el grifo
3. Llenar el vaso hasta la marca
4. Cerrar el grifo
5. Beber

❓ ¿Qué defecto tiene este algoritmo? "1. Pan 2. Tostadora 3. Mantequilla"

🌐 Paso 5: Lenguajes de programación

Hay lenguajes de tipado fuerte (Java, C) donde el tipo se declara y no cambia, y de tipado débil/dinámico como PHP, donde la variable adopta el tipo del valor. También están los compilados (generan un ejecutable) y los interpretados como PHP, que se ejecutan línea a línea.

🗣️ En este curso usaremos PHP porque su sintaxis es clara y es ampliamente usado en la web. Todo lo que aprendas sobre variables, condiciones y bucles vale para cualquier lenguaje.

📝 Paso 7: Comentarios y cómo leer errores

<?php
// Comentario de una línea
$edad = 30; // También aquí

/*
   Comentario
   de varias líneas
*/
?>

Los comentarios son ignorados por el intérprete; explican el código para humanos. Acostúmbrate a usarlos.

Cuando algo falla, PHP muestra un mensaje como:

Parse error: syntax error, unexpected '$edad' in /test.php on line 5

El número de línea y la palabra inesperada te indican dónde mirar. No tengas miedo a los errores: son pistas.

📦 Paso 8: Bloques y sentencias

Una sentencia es una instrucción completa: echo "Hola"; o $x = 5;. Las sentencias terminan con punto y coma.

Un bloque es un grupo de sentencias encerradas entre llaves { }. Se usan para que el programa trate varias instrucciones como un solo paquete, por ejemplo debajo de un if o un for.

if ($condicion) {
    // Comienza el bloque
    $contador = 1;
    echo "Dentro del if";
}   // Fin del bloque
💡 Analogía: Las llaves son como los bordes de un cajón. Todo lo que metes dentro pertenece a ese cajón, y la máquina lo abre solo cuando toca.

No te preocupes aún por qué variables son visibles dentro o fuera del bloque.

❓ ¿Cuántas sentencias hay en este bloque?

{
    $a = 1;
    $b = 2;
    echo $a + $b;
}

🔀🔀🔀 Paso 11: SWITCH

Un switch compara una variable con varios valores fijos. Es equivalente a muchos elseif.

<?php
switch ($variable) {        // ¿Qué evaluamos?
    case "valor1":          // Si $variable == "valor1"
        // código
        break;              // Salir del switch
    case "valor2":
        // código
        break;
    default:                // Opcional, si nada encaja
        // código
}
?>

Ejemplo del semáforo:

<?php
$color = "verde";
switch ($color) {
    case "rojo":
        echo "Pare";
        break;
    case "amarillo":
        echo "Precaución";
        break;
    case "verde":
        echo "Avanza";
        break;
    default:
        echo "Color no válido";
}
?>

❓ ¿Qué imprime?

<?php
$dia = 3;
switch ($dia) {
    case 1:
        echo "Lun";
        break;
    case 2:
        echo "Mar";
        break;
    case 3:
        echo "Mié";
        break;
    default:
        echo "?";
}
?>

🔁 Paso 12: Repetir con FOR

<?php
for ($i = 1; $i <= 4; $i = $i + 1) {

?>

<?php

🔄 Paso 13: WHILE

<?php
$saldo = 100;
while ($saldo > 0) {
    echo "Quedan $saldo €\n";

?>

🏁 El motor ya está en marcha

No aprendiste a programar en 14 pasos. Aprendiste algo más importante: aprendiste qué está pasando realmente cuando escribís una línea de código.

✅ Curso completado

Ahora cuando tu profesor diga "declará una variable" o "poné un if", tu cabeza ya no va a estar en blanco. Vas a saber qué está haciendo la máquina por dentro.

🧠 No eras vos, era el enfoque
Aprender a programar sin entender la RAM, los tipos, la lógica booleana o los bloques es como aprender a cocinar sin saber qué es una sartén, un horno o una cebolla. Te daban la receta y vos la seguías sin entender por qué el fuego tiene que estar medio o por qué se corta en cubitos. No es falta de inteligencia: te estaban enseñando una profesión nueva saltándose los fundamentos. Eso era lo que faltaba.

📋 Lo que ahora sabés y antes no

  • Cuando veas $nombre = "Ana"; ya no vas a pensar "es una línea que copio". Vas a imaginar un casillero en la RAM, una etiqueta pegada que dice "nombre", y dentro los números 65-110-97. Vas a entender que el signo $ no es decorativo: es la forma en que PHP dice "esto es una variable".
  • Cuando veas un if no vas a preguntarte "¿por qué a veces se ejecuta y a veces no?". Vas a saber que el bloque de llaves solo se abre si la condición es true. Punto. No hay magia.
  • Cuando escriban for ($i = 0; $i < 10; $i++) en el pizarrón, vas a poder leerlo como "empezás en 0, mientras sea menor que 10, avanzás de a uno". No es un hechizo: es un contador.
  • Cuando el código tire un error en la línea 8 no vas a entrar en pánico. Vas a ir a la línea 8, vas a mirar si falta un punto y coma o una llave, y vas a entender que el intérprete te está dando pistas porque no puede adivinar lo que quisiste hacer.

🔎 Por qué antes sentías que no entendías nada

Porque te estaban enseñando las recetas sin explicarte los ingredientes. Este curso te mostró los ingredientes: la RAM, los tipos, las condiciones booleanas, los bloques, los bucles. Todo lo demás que veas en adelante —funciones, arrays, objetos, bases de datos— son solo combinaciones más elaboradas de estos mismos ingredientes.

💡 Una aclaración importante: Este curso simplificó algunas cosas a propósito. En la realidad, PHP maneja los tipos de forma más flexible, los bucles pueden anidarse, y existen temas como el ámbito de las variables que aún no viste. Pero eso no invalida lo aprendido: es el siguiente escalón, no una corrección. La base que construiste es correcta.

🧭 Cómo seguir desde acá (sin frustrarte)

Ahora sí podés volver a tus clases o tutoriales con otros ojos:

  • Cuando el profe explique algo nuevo, preguntate: "¿qué está pasando en la RAM con esto?". Si no lo sabés, es la pregunta correcta para investigar.
  • Probá romper el código a propósito. Cambiá un true por un false, sacá una llave, poné mal el contador del for. Ver qué error sale te va a enseñar más que copiar sin errores.
  • Explicá en voz alta lo que hace cada línea como si estuvieras enseñándoselo a alguien que no sabe nada. Si no podés explicarlo, ahí hay un hueco que podés repasar.

🌱 En resumen: No salís de este curso sabiendo programar. Salís con el motor mental encendido para entender lo que programás. Eso es lo que te faltaba. El resto es práctica y tiempo. Confiá en que podés, porque la lógica ya la tenés.
